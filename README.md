GemmaClip turns video captioning from a black box into a glass box. It provides a simple Quick Caption flow for everyday users and Gemma Lab for inspecting frames, bounded audio, routing decisions, structured evidence, captions, and experiment comparisons.

## What it includes

- **Quick Caption** — upload a video and receive four grounded caption styles with the Balanced preset.
- **Gemma Lab** — inspect and configure the pipeline one stage at a time:
  `Video → Frames → Audio → Evidence → Captions → Compare`.
- **Leaderboard CLI** — a separate Python-only entrypoint that reads `/input/tasks.json` and writes `/output/results.json`.
- **Safe degradation** — valid partial captions, evidence-based fallback, progressive result writes, and bounded media processing instead of whole-batch crashes.
- **Privacy boundaries** — credentials stay server-side; run IDs are generated by the server; media and temporary audio stay outside the static frontend.

## How the pipeline works

```text
Video
  ├─ six chronological Hybrid frames
  ├─ optional bounded audio energy candidate
  └─ automatic evidence route
       ├─ visual evidence: Gemma 4 26B A4B
       └─ audio + visual evidence: Gemma 4 12B Unified
              ↓
      Gemma 4 31B caption synthesis
              ↓
      formal · sarcastic · humorous-tech · humorous-non-tech
```

Quick Caption uses six Hybrid frames—four temporal anchors plus two high-change frames—automatic audio selection, automatic routing, and the existing grounding/fallback safeguards. Gemma Lab exposes the same Python services without reimplementing pipeline logic in TypeScript.

## Model routing

The recommended public configuration is `GEMMACLIP_PROVIDER=routed_gemma`.

| Stage | Gemma role | Responsibility |
| --- | --- | --- |
| Visual evidence | Gemma 4 26B A4B | Builds structured evidence from chronological frames. |
| Audio-visual evidence | Gemma 4 12B Unified | Uses frames plus a bounded audio candidate when audio is useful and runtime permits. |
| Caption synthesis | Gemma 4 31B | Writes the requested caption styles from frames and normalized evidence. |
| Safe fallback | Google Gemma 4 31B or grounded evidence | Preserves truthful output when a provider, route, or runtime budget is unavailable. |

An AMD Cloud OpenAI-compatible deployment can replace only the audio-visual evidence role with `AMD_GEMMA_AUDIO_VISUAL_API_KEY`, `AMD_GEMMA_AUDIO_VISUAL_BASE_URL`, and `AMD_GEMMA_AUDIO_VISUAL_MODEL`. If that attempt fails, GemmaClip drops audio and continues with Google Gemma 4 31B using visual frames only.

Audio is bounded to a maximum selected window, and RMS is treated as an energy heuristic—not proof of speech. Only normalized, caption-safe audio facts may reach final captions.

See [Gemma routing](docs/GEMMA_ROUTING.md) and [Gemma audio](docs/GEMMA_AUDIO.md) for the full degradation ladder and safety rules.

## Quick start with Docker

Docker is the easiest way to run the complete web application with FastAPI, the React build, FFmpeg, and persistent run storage.

### 1. Configure runtime secrets

```bash
cp .env.example .env
```

On PowerShell:

```powershell
Copy-Item .env.example .env
```

Set at least one supported provider credential in `.env`. Never commit `.env`, API keys, authorization headers, uploaded videos, extracted audio, or generated frame artifacts.

### 2. Start the web app

```bash
docker compose -f docker-compose.web.yml up --build
```

Open [http://127.0.0.1:8000](http://127.0.0.1:8000). Useful checks:

```bash
curl http://127.0.0.1:8000/api/health
```

The health response reports storage, FFmpeg, FFprobe, the job manager, and whether a supported provider credential is configured. It does not make a live provider request or expose environment values.

To stop the service:

```bash
docker compose -f docker-compose.web.yml down
```

The named `gemmaclip-runs` volume keeps web run artifacts outside the frontend image.

## Local development

### Backend

Requirements: Python 3.12 or newer and FFmpeg/FFprobe on `PATH`.

```bash
python -m venv .venv

# macOS/Linux
source .venv/bin/activate

# PowerShell
\.venv\Scripts\Activate.ps1

python -m pip install -e ".[web,dev]"
python -m gemmaclip.web
```

The FastAPI service listens on `http://127.0.0.1:8000` by default.

### Frontend

In a second terminal:

```bash
cd web
npm install
npm run dev
```

Vite serves the React app and proxies `/api` to the local FastAPI service. Production builds use the API origin configured by `VITE_GEMMACLIP_API_BASE_URL`, or same-origin `/api` when unset.

## Leaderboard CLI

The competition surface is independent from the web app. It remains Python-only and does not require Node, React, FastAPI, or frontend assets.

Install the package:

```bash
python -m pip install -e ".[dev]"
```

Run the mounted-contract-shaped example:

```bash
python -m gemmaclip.main \
  --input examples/tasks.json \
  --output output/results.json
```

The CLI preserves the competition contract:

- reads `/input/tasks.json` at container startup;
- writes valid `/output/results.json` progressively;
- includes every requested caption style;
- returns safe fallbacks when a task or provider fails;
- keeps the 570-second application budget and 590-second container timeout;
- remains compatible with `linux/amd64`.

Build and run the competition image:

```bash
docker build -f Dockerfile -t gemmaclip-cli:latest .
docker run --rm \
  -v "$PWD/examples:/input:ro" \
  -v "$PWD/output:/output" \
  gemmaclip-cli:latest
```

The competition image is intentionally separate from `Dockerfile.web`. See [the Fireworks leaderboard guide](docs/LEADERBOARD_FIREWORKS.md) for the competition-only provider path.

## Web routes

| Route | Purpose |
| --- | --- |
| `/` | Quick Caption upload and landing page. |
| `/quick` | Processing and result view for an active Quick Caption run. |
| `/lab` | Start a manual Gemma Lab run. |
| `/lab/:runId/video` | Inspect source metadata and choose a preset. |
| `/lab/:runId/frames` | Configure and review representative frames. |
| `/lab/:runId/audio` | Inspect bounded audio selection and RMS energy. |
| `/lab/:runId/evidence` | Review route, provider, model, modality, and structured evidence. |
| `/lab/:runId/captions` | Configure styles and generate grounded captions. |
| `/lab/:runId/compare` | Save immutable snapshots and compare two experiments. |
| `/api/docs` | FastAPI OpenAPI documentation. |

Changing an upstream stage marks only its dependents stale. Completed Quick Caption runs open in Gemma Lab with their stored artifacts; they are not reprocessed unnecessarily.

## Configuration

Copy `.env.example` and set only the variables needed for the surface you are running.

| Variable | Purpose |
| --- | --- |
| `GEMMACLIP_PROVIDER` | Use `routed_gemma` for the public Gemma pipeline. |
| `GEMMACLIP_AUDIO_MODE` | `off`, `auto`, or `always`; `auto` is the recommended default. |
| `FIREWORKS_API_KEY` | Fireworks credential for configured routed roles. |
| `GOOGLE_API_KEY` or `GEMINI_API_KEY` | Google Gemma credential and fallback path. |
| `AMD_GEMMA_AUDIO_VISUAL_API_KEY` | Optional AMD Cloud credential for audio-visual evidence only. |
| `AMD_GEMMA_AUDIO_VISUAL_BASE_URL` | Optional OpenAI-compatible AMD Cloud base URL. |
| `AMD_GEMMA_AUDIO_VISUAL_MODEL` | Model ID advertised by the AMD deployment. |
| `GEMMACLIP_WEB_RUNS_DIR` | Filesystem root for web runs; `.gemmaclip/runs` locally and `/data/runs` in Docker. |
| `GEMMACLIP_WEB_MAX_UPLOAD_BYTES` | Upload limit; defaults to 200 MiB. |
| `GEMMACLIP_WEB_RUN_TTL_SECONDS` | Retention for inactive runs; defaults to 24 hours. |
| `GEMMACLIP_LOG_FORMAT` | Set to `json` for safe allow-listed lifecycle logs. |

Model role names, temperatures, runtime thresholds, and alternative provider modes are documented in [`.env.example`](.env.example), [docs/GEMMA_ROUTING.md](docs/GEMMA_ROUTING.md), and [docs/WEB_DEVELOPMENT.md](docs/WEB_DEVELOPMENT.md).

## Safety and operational boundaries

- Browser clients never receive API keys, private endpoints, authorization headers, raw base64 media, or hidden reasoning.
- Uploads are size-limited, filenames are sanitized, and server-generated run IDs prevent path traversal.
- FFmpeg and FFprobe run with argument arrays and bounded timeouts.
- Temporary audio candidates are removed after inspection or evidence generation.
- Web jobs use a single-process run lock; conflicting mutations return `409`, and active runs cannot be deleted.
- `model_generated`, `evidence_fallback`, and `deterministic_fallback` are represented as different outcomes. A fallback is never presented as a successful Gemma result when it is not.
- The web process is intentionally single-process for this release. Use an external queue and shared lock before scaling horizontally.
